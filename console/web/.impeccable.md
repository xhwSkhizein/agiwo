# Impeccable Design Context

## Project Overview

**Agiwo Console** — a control plane dashboard for an Agent SDK, built with Next.js 16 (App Router) + TypeScript + Tailwind CSS v4.

## Users

Developers and operators managing AI agent sessions, traces, and scheduled work. They need quick status visibility, the ability to drill into execution details, and configuration management for agents. The interface should evoke confidence and clarity — dense information presented cleanly.

## Brand Personality

- **Words:** Technical, precise, functional
- **Voice:** Direct and developer-oriented — no marketing fluff
- **Emotional goal:** Calm competence. Operators should feel in control, not overwhelmed.

## Aesthetic Direction

- **Theme:** Dark mode only. Deep zinc palette on near-black background.
- **Fonts:** Geist Sans (body) + Geist Mono (code/IDs/technical data).
- **Color system:** Tailwind zinc scale for surfaces and text; semantic color tokens for status badges.
- **References:** Linear, Vercel dashboard, Raycast.
- **Anti-references:** Overly decorated dashboards, heavy gradients, card-inside-card nesting, AI-slop patterns (Inter everywhere, gray sludge, bounce animations).

## Design Principles

1. **Information density without clutter** — Maximize useful signal per pixel. Status, metrics, and IDs should be scannable at a glance.
2. **Monospace for machine, sans for human** — Use Geist Mono for IDs, hashes, and technical identifiers. Use Geist Sans for labels, descriptions, and navigation.
3. **Color communicates state only** — Accent colors should indicate status (ok/error/running/idle), not decoration. Surfaces stay in the zinc palette.
4. **Dark mode done right** — Don't use pure black (#000). Use zinc-950/#09090b for background, zinc-900 for cards, zinc-800 for borders. Always add subtle borders before relying on color contrast alone.
5. **No premature complexity** — No UI component library needed. Small, composable Tailwind components with consistent prop patterns (`className`, `titleClassName`, `bodyClassName`) serve this project well.

## Key Components

- `SectionCard` — generic card with header/action slots
- `MetricCard` — label + value
- `PillBadge` — status badges
- `BackHeader` — nav header with back arrow
- `MonoText` / `MonoLink` — monospaced data display
- `ChatInputBar` / `ChatMessageItem` — chat interface
- `EmptyStateMessage` / `ErrorStateMessage` / `FullPageMessage` — state messages

## Skill Usage Notes

When using Impeccable commands on this project:
- `/audit` — scan for accessibility, dark contrast ratios, component consistency
- `/polish` — refine spacing, typography, hover states
- `/typeset` — verify font usage matches the Geist Sans/Mono system
- `/colorize` — ensure status colors maintain WCAG AA in dark mode
- `/distill` — remove unnecessary visual noise in dense data views
- `/harden` — improve error states and loading states across dashboard pages
