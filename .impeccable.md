## Design Context

### Users
Internal developers and operators use Agiwo Console to inspect agent sessions, traces, scheduler state, and configuration while debugging or running automations. The common context is a long-lived desktop browser tab used for monitoring and intervention, not marketing or demo presentation.

### Brand Personality
Calm, rigorous, dependable.

The interface should feel trustworthy and operationally clear. It should privilege legibility, state visibility, and low-friction control over novelty.

### Aesthetic Direction
A restrained dark control plane with slightly tinted neutrals, minimal accent color, and explicit status color semantics. Surfaces should feel layered but not glossy. Primary actions should stand out through contrast and clarity, while advanced controls should stay quiet until needed.

### Design Principles
1. Make system state scannable at a glance.
2. Use color for meaning, not decoration.
3. Default to simple flows and progressively disclose advanced controls.
4. Prefer inline confirmation and recovery over disruptive modal patterns.
5. Accessibility and operational resilience outrank visual flourish.

### Notes
This design context is a provisional baseline inferred from the current console use case. If you want a different brand direction, add explicit product references or tone preferences and this file should be updated.
