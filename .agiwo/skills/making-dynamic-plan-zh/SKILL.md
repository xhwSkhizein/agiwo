# 动态规划制定（OODA-COP）

动态规划是一个活的系统，能感知背景、自我纠正、自我适应 — 而不是静态的检查清单。按顺序遵循下面的三个核心引擎，然后应用适当的**适配模式**。每当新信息使之前的假设失效时，重新审视之前的引擎。

## 两种模式

此技能在两种模式下运行。根据背景选择：

| 模式 | 何时使用 | 适配指南 |
|------|--------|--------|
| **为人类** | 用户要求你帮助规划项目、策略或倡议 | [for_human/adaptation.md](for_human/adaptation.md) |
| **为 AI** | 你需要规划自己的多步任务执行 | [for_ai/adaptation.md](for_ai/adaptation.md) |

## 引擎 1：诊断 — 感知地形

对问题的不确定性水平进行分类，以选择正确的规划策略：

| 不确定性 | 策略 | 信号 |
|---------|------|------|
| **低** | 计划驱动 — 重前期设计 | 需求清晰，领域熟悉，未知因素少 |
| **中** | 价值驱动 — 按影响力排优先级 | 一些已知的未知因素，范围可能变化，信息不完整 |
| **高** | 假设驱动 — 计划 = 实验清单 | 未知的未知因素占主导，新领域，高度模糊 |

**如何诊断：**

1. 从可用背景中提取：目标、约束、截止日期、执行者、已知风险
2. 如果缺少关键信息（目标不清楚、无成功标准、执行者未知），**提出具体的澄清问题** — 不要在关键未知因素上进行假设
3. 分类为三个桶：**已知因素** / **已知的未知因素** / **疑似的未知的未知因素**
4. 从上表中选择匹配的策略
5. 产生一份**情境诊断**：复杂度水平 + 选择的策略 + 主要风险及其对冲

## 引擎 2：结构 — 分解为任务图

### 步骤 1：用 PSA 锚定范围

- **P（问题）**：我们在解决谁的什么痛点？核心需求是什么？
- **S（成功）**：定义 2–3 个可衡量的成功标准。如果执行者无法陈述它们，帮助从目标中推导。
- **A（反范围）**：明确列出范围外的内容。这可以防止范围蔓延。

### 步骤 2：建模问题类型并分解

| 问题类型 | 信号 | 分解方法 |
|---------|------|--------|
| **线性** | 清晰的因果关系，顺序步骤 | 关键路径分析 — 识别哪些延迟会级联 |
| **复杂** | 多个相互作用的部分，涌现行为 | 探测-感知-响应 — 从小实验开始，扩展有效的方法 |
| **混沌** | 无清晰的因果关系，危机模式 | 先稳定（控制伤害），然后分析 |

### 步骤 3：校准粒度

正确的细节水平取决于谁执行以及任务的耦合程度：

| 背景 | 粒度 |
|------|------|
| 专家执行者，松散耦合的任务 | 目标 + 约束 + 关键里程碑 |
| 混合团队或中等耦合 | 阶段级别，每个阶段有清晰的可交付物 |
| 新手执行者，紧密耦合的任务 | 下一步行动级别（每个步骤可在 ≤15 分钟内完成） |
| 未知执行者 | 默认为**阶段级别**，让执行者细化 |

**输出**：一份**动态工作分解** — 一个有依赖关系的任务图，而不是平面列表。

## 引擎 3：演进 — 嵌入反馈和检验点

没有反馈传感器的计划注定会失败。此引擎确保计划**监控自己**。

### 检验点类型

每个计划必须嵌入两种类型的检验点：

- **单回路**（进度检查）："我们是否按计划进行？" → 纠正行动偏差
- **双回路**（假设检查）："此计划背后的前提是否仍然有效？" → 纠正目标偏差或触发重新规划

### 预先嵌入检验点定义

对于计划中的每个检验点，明确定义：

1. **触发器**：此检验点何时触发？（在阶段 X 之后 / 在 N 天之后 / 在事件 Y 时）
2. **检查类型**：单回路或双回路
3. **要回答的问题**：具体问题，而不是模糊的"进展如何"
4. **决策规则**：根据答案采取什么行动（继续 / 调整 / 重新规划 / 上报）

### 节奏校准

将检验点频率与当前阶段相匹配：

| 阶段 | 节奏 | 理由 |
|------|------|------|
| 探索 | 每 1–2 个周期（短） | 对假设的快速反馈 |
| 执行 | 每 3–5 个周期（中） | 稳定的进度跟踪 |
| 收尾 | 阶段末回顾 | 捕获经验教训，释放资源 |

**原则**：如果环境变化速度快于检验点触发速度，你就是在盲目飞行。增加频率。

### 缓冲管理

将松弛分配为**集中缓冲池**，而不是填充每个任务：

- 根据不确定性水平估计总缓冲（高不确定性 → 更大缓冲）
- 跟踪整个计划中的缓冲消耗
- **警告阈值**：当 50% 的缓冲在完成 50% 的工作之前被消耗时，触发双回路检验点

## 何时重新规划

在以下情况下重新进入引擎 1：

- 双回路检验点揭示了无效的假设
- 主要范围变化、新约束或新利益相关者出现
- 缓冲池超过警告阈值
- 选择的策略不再适用（例如，不确定性从高降至低）
