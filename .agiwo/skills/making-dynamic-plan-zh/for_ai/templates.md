# 模板：为 AI 任务计划

## 模板 1：标准任务计划

对于中等复杂性的任务（4–10 个步骤）。

```markdown
# 任务：<简要描述>

## 诊断
- **不确定性**：低 / 中 / 高
- **策略**：计划驱动 / 价值驱动 / 假设驱动
- **关键风险**：<可能出错的地方>

## 范围
- **目标**：<成功是什么样的>
- **反范围**：<不要接触什么>

## 步骤
- [ ] 步骤 1：<操作> (依赖于：—)
- [ ] 步骤 2：<操作> (依赖于：步骤 1)
- [ ] 步骤 3：<操作> (依赖于：—，可与步骤 2 并行)
  - **检验点（单回路）**：在继续之前验证步骤 1–3 的输出
- [ ] 步骤 4：<操作> (依赖于：步骤 2、3)
- [ ] 步骤 5：<操作> (依赖于：步骤 4)
  - **检验点（双回路）**：方法是否仍然有意义？

## 验证
- <如何确认任务完成且正确>
```

## 模板 2：探索 / 研究计划

对于高不确定性的任务，其中方法不清楚。

```markdown
# 探索：<我们在调查什么>

## 假设
1. <"我相信 X 因为 Y — 通过做 Z 来测试">
2. <"我相信 A — 通过检查 B 来测试">

## 探针
### 探针 1：<小实验>
- [ ] 操作：<要做什么>
- [ ] 预期信号：<什么会确认/反驳假设>
- **结果**：<执行后填写>

### 探针 2：<小实验>
- [ ] 操作：<要做什么>
- [ ] 预期信号：<什么会确认/反驳假设>
- **结果**：<执行后填写>

## 决策点
基于探针结果：
- 如果假设 1 被确认 → 继续使用 <方法 A>
- 如果假设 1 被反驳 → 转向 <方法 B>
- 如果不确定 → 设计探针 3 针对 <特定未知>

## 执行计划
<探针完成后生成，使用模板 1>
```

## 模板 3：复杂 / 多阶段任务

对于大型任务（>10 个步骤），具有跨越多个关注点的特性。

```markdown
# 任务：<简要描述>

## 诊断
- **不确定性**：<水平>
- **策略**：<选择的策略>
- **风险**：
  1. <风险> → 缓解：<操作>
  2. <风险> → 缓解：<操作>

## 范围
- **目标**：<可衡量的结果>
- **反范围**：<边界>

## 阶段 1：<名称>
- [ ] 1.1：<操作>
- [ ] 1.2：<操作>
- **可交付物**：<此阶段产生的内容>
- **检验点**：<单回路 — 此阶段是否成功？>

## 阶段 2：<名称>
- [ ] 2.1：<操作>
- [ ] 2.2：<操作>
- [ ] 2.3：<操作>
- **可交付物**：<此阶段产生的内容>
- **检验点**：<双回路 — 假设是否仍然有效？>

## 阶段 3：<名称>
- [ ] 3.1：<操作>
- [ ] 3.2：<操作>
- **可交付物**：<此阶段产生的内容>

## 验证
- [ ] <端到端检查>
- [ ] <回归检查（如适用）>

## 缓冲
- 估计步骤：<N>
- 带缓冲（×<倍数>）：<M>
```
