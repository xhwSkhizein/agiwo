# 适配：为人类计划

完成三个核心引擎（诊断 → 结构 → 演进）后，在交付计划前应用这些人类面向的适配。

## 受众风格匹配

根据受众的认知风格调整计划的**格式和语言**：

| 受众风格 | 格式偏好 | 语言风格 |
|---------|---------|---------|
| 分析型（工程师、财务） | 数据表、逻辑链、因果图 | 精确、量化、基于证据 |
| 概念型（创意、战略家） | 愿景叙述、可能性地图、类比 | 鼓舞人心、宏观、面向未来 |
| 结构型（运营、行政） | 检查清单、流程图、SOP、甘特图 | 逐步、程序化、明确 |
| 社交型（销售、HR、领导） | 角色描述、互动地图、影响故事 | 以人为中心、关系型、激励性 |

**如何检测风格**：从用户的角色推断，他们如何描述问题（数据密集 vs 叙述 vs 流程导向），以及他们自然倾向于什么格式。不确定时，问："你更喜欢详细的分解还是高层次的概览？"

同一个计划核心可以以不同的格式呈现。选择能引起共鸣的那个。

## 授予自主权

人们对他们共同创建的计划的承诺度更高。即使目的地是固定的，也要在以下方面留出灵活性：

- **路径**：多个有效的方法 → 让执行者选择
- **节奏**：建议节奏，除非存在硬性截止日期，否则不要强制刚性时间表
- **资源分配**：提供预算/资源信封，让执行者决定分配

最低参与度：让执行者估计任务时长并确认里程碑。这将"分配的工作"转变为"拥有的工作"。

## 意义框架

用**角色和影响**的术语重新框架化任务，而不仅仅是可交付物：

| 而不是 | 框架为 |
|------|------|
| "编写 API 文档" | "作为 API 所有者，确保每个消费者可以在 <30 分钟内集成" |
| "修复入职流程" | "你正在塑造每个新用户体验的前 5 分钟" |
| "运行迁移脚本" | "你正在保护 3 年的客户数据在过渡期间" |

在计划中明确说明个别任务与最终用户影响之间的联系。

## 互动模式

当帮助人类规划时，遵循这个对话流程：

1. **先听**：让用户充分描述情况后再诊断
2. **反思回顾**：总结你的理解和诊断的不确定性水平 — 获得确认
3. **共同构建 PSA**：提议 P/S/A，邀请更正
4. **呈现草稿计划**：使用适当的模板交付结构化计划
5. **邀请细化**："你会调整什么？我遗漏了什么约束吗？"
6. **嵌入所有权**：确保用户在计划中看到自己，而不仅仅是分配给他们的任务
