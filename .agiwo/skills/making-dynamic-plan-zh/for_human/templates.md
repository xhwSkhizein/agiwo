# 模板：为人类计划

选择最适合规划场景的模板。根据需要调整部分 — 这些是框架，而不是刚性形式。

## 模板 1：项目计划（通用目的）

```markdown
# 计划：<标题>

## 情境诊断
- **复杂性**：低 / 中 / 高
- **策略**：计划驱动 / 价值驱动 / 假设驱动
- **关键风险**：
  1. <风险> → 对冲：<缓解>
  2. <风险> → 对冲：<缓解>

## 范围（PSA）
- **问题**：<谁有什么痛点>
- **成功标准**：
  1. <可衡量的结果>
  2. <可衡量的结果>
  3. <可衡量的结果>
- **反范围**：<我们将不做什么>

## 任务图

### 阶段 1：<名称>
- [ ] 任务 1.1：<描述> (依赖于：—)
- [ ] 任务 1.2：<描述> (依赖于：1.1)
- **可交付物**：<此阶段产生的内容>

### 阶段 2：<名称>
- [ ] 任务 2.1：<描述> (依赖于：阶段 1)
- [ ] 任务 2.2：<描述> (依赖于：2.1)
- **可交付物**：<此阶段产生的内容>

## 检验点
| ID | 触发器 | 类型 | 关键问题 | 决策规则 |
|----|------|------|--------|--------|
| CP-1 | 阶段 1 后 | 单回路 | 阶段 1 可交付物是否满足标准？ | 继续 / 调整阶段 2 |
| CP-2 | 阶段 2 后 | 双回路 | 原始假设是否仍然有效？ | 继续 / 从引擎 1 重新规划 |

## 缓冲池
- **总缓冲**：<X 天/小时>
- **警告阈值**：50% 消耗于 50% 完成前 → 触发双回路审查
```

## 模板 2：战略倡议

```markdown
# 战略：<倡议名称>

## 背景与诊断
- **当前状态**：<我们在哪里>
- **期望状态**：<我们想去哪里>
- **不确定性水平**：低 / 中 / 高
- **策略**：计划驱动 / 价值驱动 / 假设驱动
- **关键假设**（待验证）：
  1. <假设>
  2. <假设>

## 范围（PSA）
- **问题**：<战略差距>
- **成功标准**：<2–3 个北极星指标>
- **反范围**：<战略边界>

## 工作流

### 工作流 A：<名称>
- **所有者**：<角色/团队>
- **里程碑**：
  1. <里程碑> — 目标：<日期/条件>
  2. <里程碑> — 目标：<日期/条件>

### 工作流 B：<名称>
- **所有者**：<角色/团队>
- **里程碑**：
  1. <里程碑> — 目标：<日期/条件>

## 依赖关系
- 工作流 B.1 依赖于工作流 A.2

## 检验点
| ID | 触发器 | 类型 | 关键问题 | 决策规则 |
|----|------|------|--------|--------|
| CP-1 | <何时> | 单回路 | <具体问题> | <操作> |
| CP-2 | <何时> | 双回路 | 关键假设是否仍然有效？ | 继续 / 转向 / 停止 |

## 缓冲与应急
- **缓冲**：<数量>
- **应急计划**：如果 <风险事件>，则 <响应>
```

## 模板 3：学习 / 探索路径

```markdown
# 学习计划：<主题>

## 诊断
- **当前知识**：<我已经知道什么>
- **目标能力**：<"完成"是什么样的>
- **不确定性**：高（探索 — 假设驱动）

## 要测试的假设
1. <"我相信学习 X 将使我能够 Y">
2. <"我相信方法 A 对我的背景比 B 更好">

## 阶段

### 阶段 1：定向（<时间框架>）
- [ ] 调查景观：<要审查的资源>
- [ ] 识别关键概念和技能差距
- **检验点**：我能表达我需要学习的前 3 件事吗？

### 阶段 2：专注实践（<时间框架>）
- [ ] 深入 <主题 A>
- [ ] 构建 <小项目 / 练习>
- **检验点**：我能在没有参考材料的情况下应用概念 X 吗？

### 阶段 3：集成（<时间框架>）
- [ ] 应用于真实问题：<具体用例>
- [ ] 寻求来自 <来源> 的反馈
- **检验点（双回路）**：我的学习假设是否正确？如果不是，调整路径。

## 成功标准
1. <能力的具体演示>
2. <产生的实际输出>
```
